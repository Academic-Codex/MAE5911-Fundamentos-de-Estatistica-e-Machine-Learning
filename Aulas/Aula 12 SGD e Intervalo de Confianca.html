Parte 01 - Otimizao Numrica

1. O Problema: Quando a Soluo Analtica Falha

Em muitos problemas estatsticos, encontrar o Estimador de Mxima Verossimilhana (EMV) envolve derivar a funo de log-verossimilhana, igual-la a zero e resolver para o parmetro $\theta$. No entanto, para muitas distribuies, a equao resultante no tem uma soluo analtica ("fechada").

Estudo de Caso: A Distribuio Gama

Seja $Z_1, ..., Z_n$ uma amostra aleatria de uma populao $Z \sim \text{Gama}(\text{shape}=\theta, \text{scale}=1)$. A funo de log-verossimilhana : $$l(\theta|z_n) = (\theta-1)\sum_{i=1}^{n}\log(z_i) - \sum_{i=1}^{n}z_i - n \log(\Gamma(\theta))$$ A funo escore (derivada) : $$\frac{\partial l}{\partial \theta} = \sum_{i=1}^{n}\log(z_i) - n \psi(\theta)$$ onde $\psi(\theta) = \frac{d}{d\theta}\log(\Gamma(\theta))$ a funo digama.

O EMV, $\hat{\theta}_{MV}$, a soluo da equao $\overline{\log(Z)} = \psi(\hat{\theta}_{MV})$. No podemos isolar $\hat{\theta}$ algebricamente. Portanto, precisamos de mtodos numricos de otimizao para encontrar a estimativa.

2. Mtodos Numricos de Otimizao

Vamos comparar trs algoritmos para encontrar $\hat{\theta}_{MV}$ a partir de uma amostra simulada.

chute_inicial = mean(x)

2.1 Mtodo de Newton-Raphson (NR)

[cite_start]O Newton-Raphson um mtodo de segunda ordem, o que significa que ele usa a informao da primeira e da segunda derivada (a curvatura) para convergir rapidamente. [cite: 369-371] A regra de atualizao : $$\theta_{j+1} = \theta_j - [l''(\theta_j)]^{-1} l'(\theta_j)$$ Para a Gama, $l''(\theta) = -n \psi_1(\theta)$, onde $\psi_1$ a funo trigama.

In[4]:
theta.hat = numeric(100)
theta.hat[1] = chute_inicial
erro = 1
j = 1

# Derivadas em R: digamma() para a 1, trigamma() para a 2
score_func = function(theta, data) { sum(log(data)) - length(data) * digamma(theta) }
hessian_func = function(theta, data) { -length(data) * trigamma(theta) }

while (erro > 10^(-5)) {
  j = j + 1
  # Aplica a regra de atualizao do NR
  theta.hat[j] = theta.hat[j-1] - score_func(theta.hat[j-1], x) / hessian_func(theta.hat[j-1], x)
  erro = abs(theta.hat[j] - theta.hat[j-1])
  cat("NR Iteration:", j-1, ", theta.hat=", round(theta.hat[j], 4), "\n")
}
theta_nr = theta.hat[j]

2.2 Gradient Descent (GD)

[cite_start]O Gradient Descent um mtodo de primeira ordem que usa apenas a informao do gradiente (a primeira derivada). [cite: 355] A regra de atualizao : $$\theta_{j+1} = \theta_j + \delta \cdot l'(\theta_j)$$ onde $\delta$ a taxa de aprendizado (learning rate).

In[5]:
theta.hat = numeric(1000)
theta.hat[1] = chute_inicial
erro = 1
j = 1
delta = 0.0001 # A taxa de aprendizado precisa ser pequena e ajustada cuidadosamente

while (erro > 10^(-5)) {
  j = j + 1
  # Aplica a regra de atualizao do GD
  theta.hat[j] = theta.hat[j] + delta * score_func(theta.hat[j-1], x)
  erro = abs(theta.hat[j] - theta.hat[j-1])
  if (j %% 100 == 0) { # Imprime a cada 100 iteraes
    cat("GD Iteration:", j-1, ", theta.hat=", round(theta.hat[j], 4), "\n")
  }
  if (j >= 999) break # Critrio de parada para evitar loop infinito
}
theta_gd = theta.hat[j]
cat("GD Final Iteration:", j-1, ", theta.hat=", round(theta.hat[j], 4), "\n")

Anlise: O GD requer muito mais iteraes para convergir. [cite_start]Sua convergncia no garantida e depende criticamente da escolha da taxa de aprendizado $\delta$. [cite: 353-354] No entanto, cada iterao computacionalmente mais barata que a do NR.

2.3 Stochastic Gradient Descent (SGD)

[cite_start]O SGD uma variao do GD que aproxima o gradiente usando apenas um subconjunto de dados (um "mini-batch") em cada passo. [cite: 339-340] No exemplo da aula, usamos o caso extremo com batch size = 1, onde a atualizao feita para cada observao individualmente.

theta.hat = numeric()
theta.hat[1] = chute_inicial
erro = 1
k = 0 # Contador de pocas (passagens completas pelos dados)
j = 1 # Contador de iteraes totais
delta = 0.001 # Taxa de aprendizado para SGD

while (erro > 10^(-5) && k < 50) {
  k = k + 1 # Nova poca
  for (i in 1:n) {
    j = j + 1
    # Gradiente estocstico (usando apenas a observao x[i])
    grad_stochastic = log(x[i]) - digamma(theta.hat[j-1])

  erro = abs(theta.hat[j] - theta.hat[j-n]) 
  cat("poca:", k, ", theta.hat=", round(theta.hat[j], 4), "\n")
}
theta_sgd = theta.hat[j]

3. Resumo e Prximos Passos: Intervalos de Confiana

A escolha do mtodo de otimizao um trade-off entre custo computacional por iterao e velocidade de convergncia.

  • NR: Rpido, mas caro. Ideal para problemas de baixa dimenso.
  • GD/SGD: Lento, mas barato. Essencial para problemas de alta dimenso.
  • [cite_start]Avanados (e.g., ADAM): Mtodos como ADAM [cite: 343] so adaptativos e tentam combinar o melhor dos dois mundos, sendo o padro em muitas aplicaes modernas de aprendizado de mquina.

Agora que temos mtodos robustos para encontrar a estimativa pontual $\hat{\theta}_{MV}$, o prximo passo da inferncia estatstica quantificar a incerteza dessa estimativa. Para isso, construmos Regies (ou Intervalos) de Confiana.

Parte 02 - Intervalo de Confiana

4. Quantificando a Incerteza: Intervalos de Confiana Assintticos

Uma estimativa pontual, como $\hat{\theta}_{MV}$, nos d o valor mais plausvel para o parmetro, mas no informa nada sobre a incerteza associada a essa estimativa. Para isso, construmos Intervalos de Confiana.

[cite_start]Um intervalo de confiana com coeficiente de confiana $\gamma$ (e.g., 95%) uma regio aleatria que, antes da coleta dos dados, tem probabilidade $\gamma$ de conter o verdadeiro valor do parmetro. [cite: 273, 274] A confiana est no procedimento, no em um intervalo especfico j observado.

4.1 O Intervalo de Confiana de Wald

Um mtodo geral para construir regies de confiana assintticas baseado na Estatstica de Wald. [cite_start]Conforme a teoria (Teorema 2.4.1 e suas consequncias), se temos um estimador assintoticamente normal $\hat{\theta}_n$, podemos construir a seguinte quantidade: [cite: 565]

$$Q_n(\theta) = n(\hat{\theta}_n - \theta)^\top [V_{\hat{\theta}_n}]^{-1} (\hat{\theta}_n - \theta) \xrightarrow{D} \chi^2_p$$

[cite_start]Onde $p$ a dimenso de $\theta$ e $V_{\hat{\theta}_n}$ uma estimativa consistente da varincia assinttica. [cite: 238, 245]

[cite_start]Uma regio de confiana de nvel $\gamma$ ento o conjunto de todos os valores de $\theta$ para os quais a estatstica $Q_n(\theta)$ observada no "extrema", ou seja: [cite: 241]

$$C_\gamma(Z_n) = \{\theta \in \Theta : Q_n(\theta) \le q_{\gamma, p}\}$$

Onde $q_{\gamma, p}$ o quantil $\gamma$ de uma distribuio $\chi^2_p$.

Aplicao para a Poisson($\theta$)

Para o EMV da Poisson, $\hat{\theta}_n = \overline{Z}$, temos $p=1$ e a varincia assinttica $\theta$, que podemos estimar por $\hat{\theta}_n = \overline{Z}$. A estatstica de Wald se torna:

$$Q_n(\theta) = n(\overline{Z} - \theta)^2 / \overline{Z} \approx \chi^2_1$$

[cite_start]O intervalo de confiana derivado da inequao $Q_n(\theta) \le q_{\gamma, 1}$: [cite: 223] $$n\frac{(\overline{Z} - \theta)^2}{\overline{Z}} \le q_{\gamma, 1} \implies (\overline{Z} - \theta)^2 \le \frac{q_{\gamma, 1}\overline{Z}}{n}$$ $$\implies |\overline{Z} - \theta| \le \sqrt{\frac{q_{\gamma, 1}\overline{Z}}{n}}$$ $$\implies \overline{Z} - \sqrt{\frac{q_{\gamma, 1}\overline{Z}}{n}} \le \theta \le \overline{Z} + \sqrt{\frac{q_{\gamma, 1}\overline{Z}}{n}}$$

Este o Intervalo de Confiana de Wald para $\theta$.

5. Cobertura Nominal vs. Cobertura Real

A teoria nos diz que, quando $n \to \infty$, a probabilidade de o intervalo de Wald conter o verdadeiro $\theta$ de 95% (a cobertura nominal). Mas como ele se comporta para um n finito? A cobertura real (ou emprica) pode ser diferente.

Vamos usar uma simulao de Monte Carlo para estimar a cobertura real do intervalo de Wald para diferentes tamanhos de amostra n, replicando a anlise da aula.

Simulao: Estudando o Comportamento da Cobertura

O objetivo verificar se a "confiana de que 95% dos intervalos observados vo conter o $\theta$ certo" se sustenta na prtica.

# 1. Parmetros da simulao
set.seed(42)
# Vetor de tamanhos amostrais a serem testados
nn = c(2, 5, 10, 20, 50, 100, 200, 300, 500) 
MC = 10000 # Nmero de rplicas de Monte Carlo
gamma = 0.95 # Nvel de confiana nominal

# Vetor para armazenar a cobertura emprica para cada n
A = numeric(length(nn))
Fgamma1 = qchisq(gamma, 1) # Quantil da Chi-quadrado, ~3.84

# 2. Loop externo: itera sobre os tamanhos de amostra
for (j in 1:length(nn)) {
  n = nn[j]
  cober = numeric(MC) # Vetor para os resultados de cobertura da simulao interna
  
  # 3. Loop interno: simulao de Monte Carlo para um n fixo
  for (i in 1:MC) {
    # Geramos o theta0 aleatoriamente para testar a robustez
    theta0 = runif(1, 1, 10)
    
    # Gerando dados sob H0 (com o theta0 verdadeiro)
    x = rpois(n, theta0)
    mean_x = mean(x)
    
    # Preveno de diviso por zero se mean_x for 0
    if (mean_x <= 0) {
      cober[i] = NA
      next
    }
    
    # Calculando os limites do intervalo de Wald
    margem_erro = sqrt(Fgamma1 * mean_x / n)
    I1 = mean_x - margem_erro
    I2 = mean_x + margem_erro
    
    # Verificando se o intervalo cobriu o theta0 verdadeiro
    cober[i] = ifelse(I1 <= theta0 & I2 >= theta0, 1, 0)
  }
  
  # 4. Armazenando a cobertura emprica mdia para o n atual
  A[j] = mean(cober, na.rm = TRUE)
  cli_progress_message(paste0("n = ", n, ", Cobertura Real Atingida = ", round(A[j], 4)))
}

# 5. Visualizao dos Resultados
plot(nn, A, 
     type = "l", # 'l' para linha
     lwd = 2,
     col = "darkblue",
     ylim = c(min(A, 0.90), 1),
     xlab = "Tamanho da Amostra (n)",
     ylab = "Cobertura Emprica",
     main = "Cobertura Real Atingida vs. Tamanho da Amostra")

# Adiciona uma linha de referncia no nvel nominal
abline(h = gamma, col = "red", lty = 2, lwd = 2)
legend("bottomright", legend = c("Cobertura Emprica", "Cobertura Nominal (95%)"),

Anlise dos Resultados

O grfico gerado a principal concluso da aula sobre intervalos de confiana assintticos. Ele mostra claramente que:

  • Para amostras pequenas (n baixo): A cobertura real atingida fica significativamente abaixo da cobertura nominal de 95%. Isso significa que, se usarmos o intervalo de Wald com poucos dados, ele ser "liberal" demais e conter o parmetro verdadeiro com menos frequncia do que o prometido.
  • medida que n aumenta: A cobertura real converge para a cobertura nominal. A aproximao assinttica se torna cada vez melhor, e a promessa dos 95% de confiana se torna mais confivel.

Isso ilustra uma lio fundamental em inferncia estatstica: as propriedades tericas dos mtodos assintticos so garantias que se tornam mais precisas medida que o tamanho da amostra cresce. A simulao de Monte Carlo a nossa ferramenta para diagnosticar o quo bem essas propriedades se aplicam a um problema com um tamanho de amostra finito.
