1. Contexto Terico: Funes de Estimao

Nesta aula, foi contruido um estimador para o parmetro $\theta$ de uma populao $Z \sim \text{Pois}(\theta)$. Em vez de usar um estimador padro como a mdia amostral (que o EMV e EMM1), vamos construir e analisar um estimador derivado de uma funo de estimao mais complexa. Isso nos permite ilustrar o poder e a generalidade da teoria.

1.1 A Funo de Estimao

A funo de estimao proposta :

$$U_n(\theta) = \frac{1}{n}\sum_{k=1}^{n} u(Z_k, \theta)$$

onde a componente individual $u(Z_k, \theta)$ definida por:

$$u(Z_k, \theta) = \frac{1}{1+Z_k} - \frac{1-e^{-\theta}}{\theta}$$

[cite_start]Para que $U_n(\theta)$ seja uma funo de estimao no-viesada (Definio 2.4.3)[cite: 547], a esperana de $u(Z_k, \theta)$ deve ser zero. Vamos verificar isso. A esperana de $\frac{1}{1+Z_k}$ :

$$E_{\theta}\left(\frac{1}{1+Z}\right) = \sum_{x=0}^{\infty} \frac{1}{1+x} \frac{e^{-\theta}\theta^x}{x!} = \frac{e^{-\theta}}{\theta} \sum_{x=0}^{\infty} \frac{\theta^{x+1}}{(x+1)!}$$

Sabendo que a expanso de Taylor para $e^\theta$ $\sum_{k=0}^{\infty} \frac{\theta^k}{k!} = 1 + \frac{\theta^1}{1!} + \frac{\theta^2}{2!} + ...$, a soma $\sum_{x=0}^{\infty} \frac{\theta^{x+1}}{(x+1)!}$ igual a $(e^\theta - 1)$. Portanto:

$$E_{\theta}\left(\frac{1}{1+Z}\right) = \frac{e^{-\theta}}{\theta}(e^\theta - 1) = \frac{1-e^{-\theta}}{\theta}$$

Isso confirma que $E_{\theta}[u(Z_k, \theta)] = E_{\theta}\left(\frac{1}{1+Z_k}\right) - \frac{1-e^{-\theta}}{\theta} = 0$. [cite_start]A funo no-viesada. [cite: 487, 488, 490]

O estimador $\hat{\theta}_n$ ento definido como a raiz desta funo: $U_n(\hat{\theta}_n) = 0$.

1.2 Resolvendo $U_n$ numericamente

"Qual valor zera $U_n$?". Como a equao $U_n(\theta)=0$ no possui uma soluo algbrica simples, recorremos a mtodos computacionais.

Vamos gerar uma nica amostra aleatria e usar a abordagem demonstrada em aula para encontrar a estimativa $\hat{\theta}$ para essa amostra.

2. Implementao do Estimador em R

Como a equao $U_n(\theta) = 0$ no tem uma soluo analtica simples para $\theta$, precisamos de um mtodo numrico para encontrar a raiz. Vamos implementar as funes em R.

In[10]:
# Componente da funo de estimao
u_func = function(zk, theta) {
  1 / (1 + zk) - (1 - exp(-theta)) / theta
}

# Funo de estimao completa
un_func = function(theta, z) {
  mean(u_func(z, theta))
}

# Funo para encontrar a estimativa numericamente
# Usamos a funo 'uniroot' para encontrar a raiz de 'un_func'
# O intervalo de busca [0.01, 50]  um chute inicial razovel para theta.
calcular_estimativa = function(z) {
  # 'uniroot' precisa de uma funo de um nico argumento, por isso a definio interna
  fn_to_solve = function(theta) un_func(theta, z)
  
  # Usamos tryCatch para lidar com casos em que a raiz no  encontrada no intervalo
  result = try(uniroot(fn_to_solve, interval = c(0.01, 50))$root, silent = TRUE)
  if (inherits(result, "try-error")) {
    return(NA) # Retorna NA se a soluo no for encontrada
  } else {
    return(result)
  }
}

# Demonstrao com uma amostra
set.seed(123)
n_demo = 100
theta0_demo = 10
z_demo = rpois(n_demo, theta0_demo)
theta_hat_demo = calcular_estimativa(z_demo)

cat("Para uma amostra com n=", n_demo, "e theta0=", theta0_demo, "\n")
cat("A estimativa encontrada foi theta_hat =", theta_hat_demo, "\n")

3. Teoria Assinttica e a Quantidade Pivotal

[cite_start]O Teorema do Limite Central para Estimadores (Teorema 2.4.1) nos diz que, sob condies de regularidade, nosso estimador assintoticamente normal. [cite: 565] Para fazer inferncia (e.g., construir intervalos de confiana), precisamos de uma quantidade pivotal que convirja para $N(0,1)$.

[cite_start]Conforme as anotaes e a teoria[cite: 527, 531], essa quantidade :

$$\sqrt{n} A_n(\hat{\theta}_n)^{-1/2}(\hat{\theta}_n - \theta) \xrightarrow{D} N(0,1)$$

Onde $A_n(\theta)$ o estimador sandwich da varincia assinttica, definido como:

$$A_n(\theta) = [U_n'(\theta)]^{-2} \left( \frac{1}{n} \sum_{k=1}^{n} u(Z_k, \theta)^2 \right)$$

Primeiro, precisamos de $U_n'(\theta)$, a derivada da funo de estimao. Como $U_n(\theta)$ uma mdia, sua derivada a mdia das derivadas: $$U_n'(\theta) = \frac{1}{n}\sum_{k=1}^{n} \frac{\partial}{\partial \theta} u(Z_k, \theta) = \frac{\partial}{\partial \theta} \left( \frac{1}{1+Z_k} - \frac{1-e^{-\theta}}{\theta} \right)$$ $$U_n'(\theta) = - \frac{\theta(e^{-\theta}) - (1-e^{-\theta})(1)}{\theta^2} = - \left( \frac{e^{-\theta}}{\theta} - \frac{1-e^{-\theta}}{\theta^2} \right)$$ Note que $U_n'(\theta)$ no depende de $Z_k$, o que simplifica os clculos.

4. Verificao via Simulao de Monte Carlo

A anotao final da aula foi "Calcular U e U' e verificar via Monte Carlo". exatamente o que faremos agora. Vamos simular muitas amostras, calcular a quantidade pivotal para cada uma e verificar se a distribuio emprica se assemelha a uma Normal Padro.

In[43]:
# Funes necessrias
U1n = function(theta, z) { mean(1 / (1 + z) - (1 - exp(-theta)) / theta) }
U2n = function(theta, z) { mean((1 / (1 + z) - (1 - exp(-theta)) / theta)^2) }
U1n_deriv = function(theta) { -((exp(-theta) / theta) - (1 - exp(-theta)) / theta^2) }
An = function(theta, z) { U2n(theta, z) / (U1n_deriv(theta)^2) }

# Parmetros
n = 100
M = 5000
theta0 = runif(1, 1,100)
G = numeric(M)

# Loop de Simulao
for (i in 1:M) {
  z = rpois(n, theta0)
  theta.hat = try(uniroot(U1n, interval = c(0.1, 100), z = z)$root, silent = TRUE)
  if (inherits(theta.hat, "try-error")) { G[i] = NA; next }
  An.hat = An(theta.hat, z)
  G[i] = sqrt(n) / sqrt(An.hat) * (theta.hat - theta0)
}
G = na.omit(G)

# Visualizao
hist(G, prob = TRUE, main = paste0("Verificao do TLC (theta = ", theta0, ", n = 100)"), xlab = "G", col = "lightblue", border = "white", breaks = 30)

Concluso

O histograma da distribuio emprica da quantidade pivotal se alinha perfeitamente com a curva de densidade da Normal Padro. Isso serve como uma forte comprovao emprica de que a teoria assinttica funciona como o esperado para este estimador. A simulao valida que, para um tamanho amostral de $n=100$, podemos usar essa quantidade pivotal para realizar inferncia estatstica, como a construo de intervalos de confiana para $\theta$, com base na aproximao Normal.

5. Investigando os Limites da Teoria Assinttica

O clmax da aula foi a anlise do que acontece quando a aproximao Normal falha. A teoria "assinttica", significando que ela garantida quando $n \to \infty$. Para n finito, a qualidade da aproximao pode depender fortemente do valor do parmetro $\theta$.

Cenrio Problemtico: $\theta$ Prximo de Zero

Vamos replicar a simulao, mas agora com theta0 = 0.1, um valor muito prximo da fronteira do espao paramtrico.

Anlise: O resultado inequvoco. O histograma da quantidade pivotal G extremamente assimtrico e no se assemelha em nada curva da Normal Padro. Isso demonstra que, para n=100, a aproximao assinttica completamente inadequada quando theta muito pequeno. Usar um intervalo de confiana baseado na Normal neste cenrio seria estatisticamente invlido.

Newton-Raphson, Gradient Descent (GD) e Stochastic Gradient Descent (SGD). Estes so algoritmos de otimizao numrica que respondem pergunta: "Como encontramos a estimativa $\hat{\theta}$ na prtica, especialmente em modelos complexos?".

Nosso problema consistia em encontrar a raiz da funo de estimao $U_n(\theta)=0$. Usamos a funo uniroot() do R, que um solver eficiente para problemas de uma nica dimenso. Para problemas com mltiplos parmetros (onde $\theta$ um vetor), mtodos mais sofisticados so necessrios.

  • Mtodo de Newton-Raphson: Considerado o "padro-ouro" em estatstica para problemas com poucos parmetros. [cite_start]Ele utiliza a informao da segunda derivada (a matriz Hessiana) para convergir muito rapidamente ao mximo da verossimilhana (ou raiz do vetor escore). [cite: 370-371] [cite_start]Seu principal ponto fraco o alto custo computacional para calcular e inverter a matriz Hessiana em modelos com muitos parmetros. [cite: 379]

  • Gradient Descent (GD): Um mtodo iterativo que "desce o gradiente" da funo a ser minimizada (ou "sobe o gradiente" para maximizar). [cite_start]Ele usa apenas a informao da primeira derivada, tornando cada iterao muito mais barata que a do Newton-Raphson. [cite_start] o pilar de muitos algoritmos de aprendizado de mquina, sendo adequado para modelos com milhares de parmetros. [cite: 354]

  • Stochastic Gradient Descent (SGD): Uma variao do GD projetada para cenrios com grandes volumes de dados. [cite_start]Em vez de calcular o gradiente usando todos os dados, o SGD o aproxima usando apenas um pequeno subconjunto (um "mini-batch") em cada passo. [cite: 334, 365] Isso torna o progresso mais ruidoso, mas muito mais rpido em termos computacionais, sendo o mtodo padro para treinar redes neurais profundas e outros modelos de larga escala.

Esses mtodos formam a base da estatstica computacional e do aprendizado de mquina, permitindo-nos estimar os parmetros de modelos muito complexos onde uma soluo analtica impossvel.

Anlise: O resultado claro. Com $\theta=0.1$, a distribuio emprica da quantidade pivotal G (histograma) fortemente assimtrica direita e no se parece em nada com a Normal Padro (curva vermelha). Usar a aproximao Normal para construir um intervalo de confiana neste cenrio levaria a concluses equivocadas.

Isso acontece porque para $\theta$ pequeno, a distribuio de Poisson subjacente muito assimtrica e concentrada em poucos valores (0, 1, ...). A mdia de uma funo dessas variveis (como $U_n$) no converge "rpido" o suficiente para a Normalidade com $n=100$.

Soluo: Aumentar o Tamanho Amostral n

A teoria assinttica garante que, se aumentarmos n, a aproximao deve melhorar. Vamos verificar isso repetindo a simulao para theta0 = 0.1, mas agora com n = 1000.

Concluso Final da Aula

A anlise de um estimador no trivial para o parmetro da Poisson nos permitiu explorar o ciclo completo da inferncia estatstica e aprender uma lio fundamental:

  1. A teoria das Funes de Estimao nos d um framework poderoso e unificado para criar e analisar estimadores.
  2. O Teorema do Limite Central para Estimadores e o Teorema de Slutsky fornecem as bases tericas para a inferncia assinttica, permitindo-nos construir quantidades pivotais.
  3. As Simulaes de Monte Carlo so uma ferramenta indispensvel para o estatstico moderno. Elas nos permitem:
    • Verificar se a teoria assinttica se aplica bem em cenrios prticos com amostras finitas.
    • Identificar os pontos de falha da teoria (e.g., parmetros prximos da fronteira, amostras pequenas).
    • Ganhar intuio sobre o comportamento dos procedimentos estatsticos.

A concluso que a teoria e a simulao no so excludentes; elas so parceiras. A teoria nos guia sobre o que esperar, e a simulao nos mostra se essa expectativa realista em uma situao particular.
